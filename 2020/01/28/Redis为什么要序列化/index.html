Redis为什么要序列化
Created 2020-01-28| Updated 2020-01-28|面试redis
Post View:|comments:

一 目的

序列化最终的目的是为了对象可以跨平台存储，和进行网络传输。而我们进行跨平台存储和网络传输的方式就是IO，而我们的IO支持的数据格式就是字节数组。

本质上存储和网络传输都需要经过把一个对象状态保存成一种跨平台识别的字节格式，然后其他的平台才可以通过字节信息解析还原对象信息。

二 序列化方式对比

1.JdkSerializationRedisSerializer

redis的默认方式是JdkSerializationRedisSerializer，使用JDK提供的序列化功能。

1.1 优点

JdkSerializationRedisSerializer的优点是在反序列化时不需要提供类型信息(class)

1.2 缺点

缺点是需要实现Serializable接口，还有序列化后的结果非常庞大，是JSON格式的5倍左右，这样就会消耗redis服务器的大量内存。

在RDM工具中查看k-v值时会出现“乱码”，不方便查看。

2.Jackson2JsonRedisSerializer

使用Jackson库将对象序列化为JSON字符串。

2.1 优点

优点是速度快，序列化后的字符串短小精悍，不需要实现Serializable接口。

2.2 缺点

但缺点也非常致命，那就是此类的构造函数中有一个类型参数，必须提供要序列化对象的类型信息(.class对象)。 通过查看源代码，发现其只在反序列化过程中用到了类型信息。
