二 实操

1.下载事务协调器

这里是直接上GitHub上面下载，关于LCN更多的资料你可以参考[这里的第二点]（http://facai1983.top/2020/01/06/%E4%BD%BF%E7%94%A8LCN%E5%AE%9E%E7%8E%B0%E5%88%86%E5%B8%83%E5%BC%8F%E4%BA%8B%E5%8A%A1%E4%B8%80%E8%87%B4%E6%80%A71/）这里我用的是4.1.0.RC1版本的。

2.配置redis

因为要存储事务组和补偿事务，你要需要一个配置了redis的虚拟机。

3.配置事务协调器

接下来就是找到application.properties文件，配置他。在这里面主要需要配置数据库/redis地址/注册中心。

4.配置tx-client

使用 LCN 做分布式事务管理时，服务需要知道事务协调器的 URL（客户端与事务协调器是通过 URL 来建立连接的），我们需要实现一个接口 TxManagerTxUrlService，这个接口实现类可以使用独立应用定义，在微服务应用中引入 。这里创建了一个tx-tran-manager-service负责这个模块。

java

需要注意，因为这个项目最终会注入到其他项目中，所以将tx.manager.url放到该项目下的resource文件夹下是不可行的，不会被解析，需要放到order及menu client三个实际调用他的项目中。

5.使用LCN本身提供的注解实现事务一致性

注意：这里的@Transactional注解是Spring提供的声明式事务管理方式，当你的方法中抛出异常时，它会将事务回滚，数据库中的数据将不会改变，也就是回到进入此方法前的状态只需要添加到具体操作数据库的方法上。
